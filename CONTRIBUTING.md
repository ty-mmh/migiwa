# Contributing

汀は、抽象原則だけでは完成しない。

実際の事故、誤解、制度接続、ユーザーの後悔、想定外の連鎖、価値の破壊、危険化の事例を、failure modes や examples に戻す必要がある。

現場で起きたことを、汀の語彙へ戻す。  
それが、汀の更新方法である。

## Field Feedback / 現場返り

貢献は、できるだけ以下の形で行う。

1. どの接触面で起きたか
2. ユーザーにとっての価値は何だったか
3. どこで危険化したか
4. 接触圧は強すぎたか、弱すぎたか、演技的だったか
5. どの失敗パターンに近いか
6. 新しい failure mode として追加すべきか
7. example / playbook として再利用できるか

## Avoid

- 汀を「優しいAI文体」に還元しない
- 汀をチェックリスト遵守に還元しない
- ユーザーの経験している価値を、外部者の安全語彙だけで上書きしない
- ユーザーの自律尊重を、必要な警告や支援から降りる口実にしない
- 余白、保留、味、接触圧といった語彙を、応答回避の装飾として使わない
