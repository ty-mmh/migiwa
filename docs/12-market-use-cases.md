# Market Use Cases / 市場用途

汀そのものを売るのではなく、汀を使ったレビュー、設計、監査、応答時プロトコル調整を提供する。

## Candidate service names

- Migiwa Review
- Contact Pressure Audit
- Human-Contact Evaluation
- AI Output Review
- AI Deployment UX Review
- Migiwa Response Protocol Design
- Runtime Prompt Audit
- Output Placement Review
- Human-mediated Action Review

| Priority | Area | First product shape |

## Runtime applications

汀は、出力後レビューだけでなく、応答生成時の薄い実行層としても使える。

候補:

- **Runtime prompt audit**  
  既存の system prompt / developer prompt / custom instruction が、接触圧を上書き・遺棄・過剰摩擦へ寄せていないかを見る。

- **Migiwa Response Protocol design**  
  低リスク会話では軽く、現実世界に出る出力では必要な摩擦を入れる応答設計を作る。

- **Output placement review**  
  AI出力が、メール、投稿、契約、制度、関係、判断材料としてどこに置かれうるかを見る。

- **Human-mediated action review**  
  AIが直接行動しなくても、ユーザーを通じて現実世界の操作を起こす箇所を点検する。

## Core market difference

汀は、安全と有用性の両方をレビューする。

強すぎると上書き。  
弱すぎると遺棄。

この両方を見ることが、汀の差分である。

## What not to sell as Migiwa

- 汀っぽい文体。
- 一般的な拒否ポリシー。
- すべての会話を安全側へ重くするガードレール。
- ユーザーを疑うリスク検知フレームワーク。
- プロンプト芸としての便利テンプレート。

汀は、AIの出力がユーザーの世界に置かれる直前の接触圧を見る。
